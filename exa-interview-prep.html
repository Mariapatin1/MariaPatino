Exa Deep Dive

What they actually build
A search API that lets developers add web search to AI apps and agents. Unlike Google, Exa built their own web index from scratch using neural embeddings — it understands the meaning of your query, not just keywords. Their model is trained on "link prediction" — given a description, it predicts which URLs would naturally be linked. This returns cleaner, higher-quality results for AI systems.
Key products
  • Search API — core product, semantic search for developers
  • Contents API — fetches and cleans web page content (no HTML junk)
  • Find Similar API — finds pages similar to a given URL
  • Websets (new, ~2025) — lets business teams build curated datasets from the web using AI search, aimed at sales/marketing/research
Founders
Will Bryk (CEO) and Jeff Wang — both from Harvard, math/CS backgrounds. Started as Metaphor Systems, rebranded to Exa in mid-2023. Y Combinator S21 batch.
Funding
~$22-25M total raised. Series A of $17M led by Lightspeed Venture Partners (early 2024). YC-backed. Angel investors from OpenAI and Google Brain circles.
Team size
Small — roughly 15-30 people. Engineering-heavy, lean startup. Based in San Francisco.
Who uses it
AI developers building agents and LLM-powered tools. Integrated with LangChain, LlamaIndex, CrewAI. Available as an MCP tool for Claude. Used by startups building research tools, coding assistants, news aggregators. Websets expanding into sales/marketing teams.
Why they matter
As AI agents become mainstream, they all need a way to search the web. Exa is positioning to be that critical layer in the AI stack. They're NOT a Google wrapper — they built their own index with a novel approach. Competitors include Tavily, Perplexity (more consumer-focused), and Brave Search API.
